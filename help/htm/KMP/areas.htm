Area

Areas are zones that cover a certain portion of the track. Like Routes, area functionality depends on whatever thing is linked to it.

NOTE: Looks like the AREA section of the MKWII and MK7 kmp are almost the same, so this information can be interchangeable.

Parameters

Parameters are displayed in the entry list:

  • Shape: Defines the shape of an area.
    • 0 - Box: Area is a box.
    • 1 - Cylinder: Area is a cylinder.
  • Type: Type of the area. Only 4 area types are used unlike MKWII which has 10 different area types:
    • 0 - Camera: Defines the Camera Index that should be used when the player is inside the area. This type of area doesn't use any special Settings nor is linked to any RouteID. On the other hand it's the only area type that uses the CAMEIndex parameter.
    • 1 - Particle Effect: Shows the slot specific particles on screen if the player is inside this area type. The way the particles are shown depend on the area Settings. (It is recommended to check original tracks as the settings depend on the slot.)
    • 3 - Moving Road: All the geometry using the Moving Road kcl type inside the area will push the player in the direction of the route linked by the RouteID parameter.
    • 11 - Sound Effect: Performs an specific sound effect if the conditions are met, sometimes used along with Sound Effect objects. The behaviour of this area type depends of the Settings1 parameter. RouteID can also be used to define the sound effect boundaries depending on the Settings. (It is recommended to check original tracks as the settings depend on the slot.)
  • CAMEIndex: Used with Type 0, defines the Camera Index that should be used when the player is inside the area. (Looks like some Type 1 areas in original nintendo tracks also have this value set, however it is probably a mistake by the developers or a dropped feature).
  • Priority: When two areas intersect, the area with the highest Priority will be used (almost never used).
  • PositionXYZ: Position of the center of the area.
  • RotationXYZ: Rotation of the area.
  • ScaleXYZ: Scale of the area, depending on the shape, the area will be displayed as a box or cylinder in the editor. (NOTE: since the editor is only 2D, keep in mind that for boxes, areas extend 1000 * scaleY units upwards only and for cylinders, 500 * scaleY units upwards only, so to make sure the area covers the entire section, change the PositionY to 0 or less.)
  • Settings1/2: The meaning of these parameters depend on the Type of the area. (It is recommended to check original tracks as the settings depend on the slot.)
  • RouteID: Route linked to the AREA. How the route is used depends on the area Type.
  • EnemyID: Leftover parameter for area type 4 in MKWII, never used in MK7 (always -1).
  • Unknown: The purpose of this parameter is unknown, probably padding (always 0000).

Tips

  • Make sure there isn't any place on the track outside replay areas, so they are played correctly. It is okay to overlap replay areas as the game only checks for the next area until you leave the current one.
  • Since the editor is only 2D, keep in mind that for boxes, areas extend 1000 * scaleY units upwards only and for cylinders, 500 * scaleY units upwards only, so to make sure the area covers the entire section, change the PositionY to 0 or less.
  • Some Sound effect and particle effect Type areas with certain Settings need an specific Object to be added to the track or inside the area, otherwise the game may crash.
