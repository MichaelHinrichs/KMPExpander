Objects

Objects are entities that can be interacted with. This includes item boxes, coins, moving geometry, start grid, etc.

Every track needs at least two objects to load:

  • Start Grid: This object represents the start position in normal tracks (the same way Start Position points do in battle tracks). Unknown1 value for this object is 0054. IMPORTANT: For some reason the start grid rotation is offset by 180º, so when setting its RotationY make sure you take this into account. The center of the start grid is where the player spawns in Time Trials so it should be placed just before the finish line.
  • Any Skybox Variant: This object represents the sky box and should be put in the center of the track and its size can be adjusted. Some track slots need an specific skybox variant, so try to use the original track skybox if possible.

Parameters

Parameters are displayed in the entry list:

  • ObjectType: The type of the object. When clicking this parameter in the entry list a window will appear where you can select any of the 217 MK7 objects.
  • Unknown1: The purpose of this parameter is unknown. Seems to be related to how objects render. It is recommended to check original tracks and copy this value from existing objects.
  • PositionXYZ: Position of the object.
  • RotationXYZ: Rotation of the object.
  • ScaleXYZ: Depending on the ObjectType this can be a multiplier for the object size (example: 1.5 times bigger) or the amount of units the object extends (example: 900 units in the X direction) the second being used in underwater boxes.
  • RouteID: For objects that support routes, the linked Route ID. -1 if no route is linked to the object.
  • Settings1/2/3/4/5/6/7/8: ObjectType specific settings, default value being 0. Each one of the settings field has an special meaning. It is recommended to check original tracks to see what they mean, most of the cases being a timing/speed value.
  • Visibility: Controls the visibility of objects in different game modes. It is a bitmap of 3 bits, the meaning of each individual bit is unknown. The default value for this field is 7.
  • Unknown2/3: The purpose of this parameter is unknown. Unknown2 default value is FFFF; Unknown3 deafult value is 0000.

Tips

  • Some objects need extra bcmdl and/or kcl files in the szs to load, make sure you have included all of them. You can use this tool to extract all the files from an szs and repack them afterwards.
  • Always check original tracks for object specific Unknown1 and Settings. A wrong setting could cause the object to malfunction or crash.
